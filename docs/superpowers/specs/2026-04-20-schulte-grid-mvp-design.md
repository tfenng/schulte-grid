# 舒尔特方格 MVP 设计

## 1. 目标

在当前规划仓库中先落地一个可玩的 Windows 桌面版 MVP，跑通 `首页 -> 模式选择 -> 训练 -> 结算 -> 历史记录` 的完整闭环。

本阶段优先目标：

- 可启动并运行 Tauri 桌面应用
- 可完成一局舒尔特方格训练
- 可保存并查看历史记录
- 重启后历史记录仍可读取

本阶段不做：

- 每日计划、排行榜、成长报告
- 颜色干扰、倒序、奇偶数、记忆闪现
- 通知、自动更新、复杂桌面集成功能

## 2. 范围

### 页面范围

- 首页：显示开始训练入口、最近最佳成绩、历史记录入口
- 模式选择页：选择棋盘尺寸，首版开放 `3x3`、`4x4`、`5x5`
- 训练页：渲染棋盘、显示当前目标、计时、暂停/继续、退出
- 结算页：显示完成时长、平均点击间隔、是否刷新最佳成绩、再来一局
- 历史记录页：显示历史局结果列表

### 玩法范围

- 仅支持 `正序` 规则
- 点击错误数字时采用 `无效点击`：不推进、不计错、不罚时
- 单局完成条件为按顺序点击完全部数字

## 3. 技术方案

采用“前端优先，桌面最薄”的实现路径。

- `src/` 负责 React 页面、训练引擎、状态流转与本地持久化
- `src-tauri/` 先只承担 Tauri 宿主、窗口与打包能力
- 历史记录优先保存在前端本地存储，确保 MVP 尽快跑通

原因：

- 当前目标是尽快交付可玩版本
- 训练逻辑在 TypeScript 中实现更快，便于迭代
- 后续若需增强统计或宿主能力，再把边界逐步下沉到 Rust

## 4. 模块划分

- `src/pages/`：首页、模式选择、训练、结算、历史记录
- `src/components/`：棋盘、顶部信息栏、结果卡片等复用组件
- `src/lib/game/`：棋盘生成、目标推进、计时结果计算
- `src/lib/storage/`：训练结果读写封装
- `src/types/`：`GameConfig`、`GameSession`、`GameResult`
- `src-tauri/`：Tauri 入口与基础配置

## 5. 数据模型

- `GameConfig`：棋盘尺寸、规则类型、错误策略
- `GameSession`：当前棋盘、目标数字、开始时间、暂停状态、点击进度
- `GameResult`：完成时间、平均点击间隔、棋盘尺寸、完成时间戳

MVP 中固定：

- 规则类型为 `ascending`
- 错误策略为 `ignore`

## 6. 验收标准

- 能在本地启动 Tauri 开发环境
- 能完成 `3x3`、`4x4`、`5x5` 任一模式的正序训练
- 完成后自动进入结算页
- 结算结果可写入历史记录
- 重启应用后历史记录仍然可见
